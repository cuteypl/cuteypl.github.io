关于npm run dev报错的那些吐血操作(一)

1、npm run dev时报解析css错误

1.1 截图

Tips：原因是没有安装css-loader依赖。

1.2 解决

如果是 常规 的，执行 npm install stylus-loader css-loader style-loader --save-dev 安装依赖就行。
如果是 less 的，执行 npm install less less-loader --save-dev 安装依赖就行。
如果是 sass 的，执行 npm install sass sass-loader --save-dev 安装依赖就行。或者（$npm intall sass-loader --save ; $npm install node-sass --save）

2、npm run dev时报错如下截图

2.1 截图

Tips：原因可能是webpack与webpack-dev-server及其他一些依赖包的版本兼容性问题。

2.2 解决

到github上查阅webpack、webpack-dev-server等版本更新描述，一般会声明最小兼容版本；
如：minimum require wepback version is 4.0.0 / minimum required Node.js version is 8.9.0等；
根据需要卸载，重新安装指定版本，以达到兼容(npm uninstall xxx --x参数，npm install xxx@version版本号 --x参数)。或者在package.json文件上直接修改版本号，再npm install。

3、npm run dev时提示如下

3.1 描述

vulnerabilities run npm audit fix to fix them

3.2 原因

首先这是npm@5.10.0 and npm@6才有npm audit这个命令。
该命令用于执行项目依赖关系树的即时安全性审查。审核报告包含有关依赖项中的安全漏洞的信息，可以通过审核报告提供的简单易用的npm命令和建议能修复漏洞，以便进一步排除故障。
为了最大限度地防止不安全的代码，建议使用npm @ 6+。通常运行npm audit fix命令就可以修复漏洞。
